# 論文選題方法論：從文本細讀出發

## 核心理念

**論文選題應該從文本細讀出發，而不是理念先行。**

所有觀點都應該來自原文的細讀，而不是先入為主，使用固定的邏輯框架。越是容易想到的邏輯框架，越沒有觀點，實際上只是在重複語言和思維定式中的邏輯框架。

### 兩種方法的對比

#### ❌ 錯誤方法：理念先行

```
1. 先確定研究論題（抽象理念）
   ↓
2. 根據論題形成分論點（套用現成框架）
   ↓
3. 去找 Primary Sources（選擇性找證據）
```

**問題**：
- 先有觀點，再找證據（容易選擇性摘錄）
- 忽略與觀點矛盾的證據
- 強行用文本證明先入為主的觀點
- 缺乏來自文本的獨特性
- 只是重複現成的邏輯框架，沒有真正的觀點

#### ✅ 正確方法：從文本出發

```
1. 整體閱讀 + 興趣 → 關鍵詞（模糊但有方向）
   ↓
2. 關鍵詞 → 尋找 Primary Sources（回到原文）
   ↓
3. Primary Sources → 細讀沉思 → 初步歸類（代表思考雛形）
   ↓
4. Primary Sources + 初步分論點 → 完善研究論題（形成探究問題）
   ↓
5. 探究問題 → 回過頭精煉分論點（迭代完善）
```

**特點**：
- 從模糊的抽象到據此尋找具象的文本
- 通過文本去完善抽象，讓抽象的思維更加具有來自文本的獨特性
- 形成「問題 ↔ 證據 ↔ 分論點」的循環
- 可以回溯到前面的步驟進行調整

## 正確流程詳解

### 第一步：從整體閱讀到關鍵詞

**目標**：根據整體閱讀和個人興趣，確定一個具象的關鍵詞

#### 關鍵詞的選擇標準

**✅ 好的關鍵詞特徵**：
- **具象**：能直接指向原文中的具體證據
- **可操作**：方便據此找到直接相關的 primary source
- **有方向但不確定**：模糊但有探索方向

**✅ 好的關鍵詞範例**：
- 「數字」：可以在原文中直接找到「十二」「二十四」「三百六十五」等具體數字
- 「顏色詞」：可以找到「紅」「綠」「白」等具體顏色描寫
- 「稱呼語」：可以找到不同人物對同一人的不同稱呼方式

**❌ 需要避免的關鍵詞**：
- 「主題」：太抽象，無法直接找到 primary source
- 「人物形象」：太寬泛
- 「藝術手法」：太籠統
- 「階級批判」：這是抽象理念，不是從文本出發

#### 核心原則

關鍵詞最好能夠比較方便據此找到直接相關的 primary source，而不是一些大而模糊的理念，這樣很難去據此縮小並尋找原文中的 primary source。

---

### 第二步：回到原文尋找證據

**目標**：根據關鍵詞回到原文，收集可摘錄的原文證據

#### 什麼是 Primary Source？

Primary source 必須是：
- **可摘錄的原文證據**：不是抽象概念，而是可以直接引用的完整句子/段落
- **有具體位置**：標註具體位置（回目、章節）
- **有上下文**：不能斷章取義，要有完整的語境

#### 收集標準

- **數量建議**：收集 15-30 處相關證據，確保有足夠的文本支撐
- **類型多樣**：可以是對話、描寫、結構性安排等
- **完整性**：同一關鍵詞出現多次，要判斷哪些是「有效證據」

#### 示例：尋找「數字」相關的 Primary Sources

```
第一回：甄士隱夢中見「一僧一道」，賈雨村看到「十二釵」
第五回：太虛幻境「十二金釵」正冊、副冊、又副冊
第三十九回：劉姥姥說「這園子裡頭，就是這一年裡頭，也該有三百六十五個窟窿」
...（繼續尋找更多）
```

---

### 第三步：細讀與歸類

**目標**：細讀這些 primary sources，大致歸類為 2-4 類（通常是 3 類）

#### 歸類的性質

這個分類**不一定**按 primary source 本身的「客觀性質」（如「數字 12」「數字 24」「數字 365」），而是根據你的細讀思考，將這些 primary sources 大致歸為幾類。這個分類代表了你的思考雛形，會形成論文的分論點。

#### 框架的使用與優化

**可以使用常見框架，但必須優化**：

可以使用常見的小說要素（人物、情節、主題），但必須用你的獨特思考去補充、優化，使其脫離「一般框架」的感覺。

**❌ 一般框架（套用現成分類）**：
- 人物、情節、主題

**✅ 優化後的框架（注入文本獨特性）**：
- 人物稱呼語的層次變化（發現：同一個人，不同人稱呼不同）
- 數字與命運的隱喻關聯（發現：十二金釵的「十二」與其他「十二」形成對應）

#### 如何判斷是否「套用框架」？

- **問自己**：這些分類是否真的來自你讀到的文本？還是你預先知道的？
- **檢查標準**：如果分類只是「人物、情節、主題」這種空泛的標籤，沒有注入從文本中發現的具體特點，那就是套用框架

#### 示例：數字的分類

```
分類一：結構性數字（十二金釵、十二回等，與小說結構相關）
分類二：時間性數字（三百六十五、二十四節氣等，與時間概念相關）
分類三：象徵性數字（一僧一道、三生石等，與宗教哲學相關）
```

### 第四步：從證據到問題

**目標**：根據 primary sources 和初步分論點，回過頭完善研究論題（形成探究問題）

#### 回溯式完善

不是一開始就確定論題，而是基於已收集的資料和初步思考來完善。論題應該是探究問題性質的。

#### 探究問題的特點

好的研究論題應該是一個**可以探究的問題**，而不是一個已經確定的結論。

**❌ 錯誤示例**：
- 「紅樓夢中數字暗示人物命運」（這是結論，不是問題）

**✅ 正確示例**：
- 「紅樓夢中數字如何暗示人物命運？」（這是探究問題）
- 「紅樓夢中數字的結構性安排如何暗示人物命運？」（更具體）

#### 完善過程

```
從模糊的關鍵詞「數字」
  ↓
到收集的 primary sources 和初步分類
  ↓
完善為探究問題：「紅樓夢中數字的結構性安排如何暗示人物命運？」
```

---

### 第五步：迭代精煉

**目標**：根據完善的探究問題，回過頭精煉分論點

#### 迭代完善的機制

形成探究問題後，回過頭來精煉分論點。這是一個循環過程：

```
問題 ↔ 證據 ↔ 分論點
```

**可以回溯到前面的步驟**：
- 在形成探究問題後，發現之前的關鍵詞需要調整 → 回到第一步
- 在精煉分論點時，發現 primary sources 不夠 → 回到第二步補充
- 在完善論題時，發現歸類不合理 → 回到第三步重新歸類

#### 最終結構

最終形成：
- **研究主題**（探究問題）
- **分論點**（通常 3 個，來自文本細讀的歸類）
- **支持分論點的 primary sources**（每個分論點都有相應的原文證據支撐）

#### 示例：最終論文結構

```
研究主題：紅樓夢中數字的結構性安排如何暗示人物命運？

分論點一：結構性數字（十二金釵）與小說的整體架構
  → Primary Sources：第一回、第五回相關描寫

分論點二：時間性數字（三百六十五）與人物的時間感知
  → Primary Sources：第三十九回等相關描寫

分論點三：象徵性數字（一僧一道）與命運的哲學隱喻
  → Primary Sources：第一回、相關回目描寫
```

## 核心原則總結

### 1. 從模糊抽象到具象文本

```
模糊抽象（關鍵詞）
    ↓
具象文本（Primary Sources）
    ↓
完善抽象（帶有文本獨特性的思維）
```

關鍵是「具象文本」作為橋樑，讓抽象思維不是憑空產生，而是從文本中「長出來」。

### 2. 具象性優先

- 關鍵詞要具象，便於找到 primary source
- Primary source 必須是可摘錄的原文證據
- 所有觀點都要有具體的文本支撐

### 3. 框架優化，而非套用

- 可以使用常見框架，但要用獨特思考去補充、優化
- 讓分類脫離「一般框架」的感覺
- 注入從文本中發現的具體特點

### 4. 迭代完善，可以回溯

- 研究是一個動態過程，不是線性的
- 可以回到任何前面的步驟進行調整
- 形成「問題 ↔ 證據 ↔ 分論點」的循環

### 5. 避免理念先行

- 不要先有觀點再找證據
- 不要套用現成的邏輯框架
- 不要選擇性摘錄，忽略矛盾證據
- 所有觀點都要來自原文細讀

本方法適合 **9 年級以上**的學生，幫助他們：

- 學會正確的論文選題方法
- 避免理念先行的錯誤
- 培養文本細讀和獨立思考的能力
