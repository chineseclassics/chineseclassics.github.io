## ADDED Requirements

### Requirement: 絳珠系統（核心資源）
系統 SHALL 實現以下功能：
絳珠是遊戲的核心資源，用於澆灌所有花魂，統一資源不分類型。

#### Scenario: 絳珠獲取
- 絳珠通過完成 pearl 類記憶獲得
- 完成記憶後，根據記憶重要性和答題表現計算絳珠數量
- 絳珠數量範圍：5-15 顆（根據記憶重要性）
- 答題表現影響最終獲得的絳珠數量（時間和答錯懲罰）

#### Scenario: 絳珠消耗
- 澆灌花魂消耗絳珠（每次澆灌消耗固定數量，如 5 顆）
- 絳珠不足時，澆灌按鈕禁用
- 絳珠消耗後立即更新顯示

#### Scenario: 絳珠顯示
- UI 狀態列顯示當前絳珠數量
- 絳珠數量使用繁體中文標籤：「絳珠：X」
- 絳珠數量實時更新
- 獲得絳珠時顯示動畫或提示（可選）

### Requirement: 靈石系統（次要資源）
系統 SHALL 實現以下功能：
靈石是次要資源，用於解鎖建築，獨立於絳珠獲取。

#### Scenario: 靈石獲取
- 靈石通過完成 stone 類記憶獲得
- 完成記憶後，根據記憶重要性和答題表現計算靈石數量
- 靈石數量根據記憶重要性而定（數量待定，需要平衡）
- 答題表現影響最終獲得的靈石數量（時間和答錯懲罰）

#### Scenario: 靈石消耗
- 解鎖建築消耗靈石（如解鎖瀟湘館需要 20 靈石）
- 靈石不足時，解鎖建築按鈕禁用
- 靈石消耗後立即更新顯示

#### Scenario: 靈石顯示
- UI 狀態列顯示當前靈石數量
- 靈石數量使用繁體中文標籤：「靈石：X」
- 靈石數量實時更新
- 獲得靈石時顯示動畫或提示（可選）

### Requirement: 資源獲取機制
系統 SHALL 實現以下功能：
所有資源都通過記憶收集獲得，沒有其他來源（MVP 階段）。

#### Scenario: 資源獲取流程
- 玩家完成記憶答題（3 題全部答對）
- 系統計算資源獎勵（根據時間和答錯次數）
- 資源直接添加到玩家資源總量
- UI 更新顯示新資源數量

#### Scenario: 資源獲取驗證
- 只有完成記憶答題才能獲得資源
- 資源獲取數量必須正確計算
- 資源獲取後必須正確更新到遊戲狀態

### Requirement: 資源不足處理
系統 SHALL 實現以下功能：
當資源不足時，系統應提供清晰的提示。

#### Scenario: 按鈕禁用提示
- 資源不足時，對應的行動按鈕禁用（視覺上變灰）
- 滑鼠懸停時顯示提示文字：「需要 X 顆絳珠」或「需要 X 顆靈石」
- 提示文字清晰顯示所需資源數量

#### Scenario: 資源獲取引導
- 資源不足時，可以顯示提示引導玩家收集更多記憶
- 提示不強制，僅作為建議
- 提示可以指向未解鎖的記憶
