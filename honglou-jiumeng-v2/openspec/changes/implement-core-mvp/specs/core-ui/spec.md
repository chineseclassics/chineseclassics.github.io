## ADDED Requirements

### Requirement: 狀態列 UI
系統 SHALL 實現以下功能：
頂部狀態列顯示遊戲的核心狀態信息。

#### Scenario: 狀態列內容
- 節氣顯示：當前節氣名稱（如「立春」）
- 行動力顯示：當前行動力 / 最大行動力（如「4 / 4」）
- 絳珠顯示：當前絳珠數量（如「絳珠：15」）
- 靈石顯示：當前靈石數量（如「靈石：20」）

#### Scenario: 狀態列更新
- 狀態變化時，UI 立即更新
- 狀態顯示清晰易讀
- 狀態列始終可見，不隨頁面滾動隱藏

### Requirement: 記憶列表 UI
系統 SHALL 實現以下功能：
記憶列表顯示所有記憶，包括已解鎖和未解鎖的記憶。

#### Scenario: 記憶列表顯示
- 顯示所有記憶，每個記憶標註「第 X 回的記憶 - 回憶名稱」
- 未解鎖記憶顯示為灰色，已解鎖記憶顯示為綠色
- 記憶列表可以滾動（如果記憶較多）
- 記憶列表始終可見

#### Scenario: 記憶操作
- 未解鎖記憶可以點擊，進入答題界面
- 已解鎖記憶顯示「已解鎖」標籤，不可點擊
- 點擊記憶後，以模態框形式顯示答題界面

### Requirement: 答題界面 UI
系統 SHALL 實現以下功能：
答題界面清晰顯示題目、選項、時間限制和答錯次數。

#### Scenario: 答題界面佈局
- 模態框形式顯示，覆蓋整個頁面
- 顯示記憶標題：「第 X 回的記憶 - 回憶名稱」
- 顯示題目序號：「題目 1 / 3」
- 顯示題目文字和 4 個選項
- 顯示時間倒計時和答錯次數

#### Scenario: 答題界面交互
- 點擊選項後，立即檢查答案
- 答對後顯示成功提示，自動進入下一題
- 答錯後顯示錯誤提示，可以重試
- 超時後顯示超時提示，該題目無法獲得資源

#### Scenario: 答題界面關閉
- 提供關閉按鈕（×）
- 關閉時停止計時器
- 關閉時不保存進度（答題必須完成才能解鎖記憶）

### Requirement: 行動面板 UI
系統 SHALL 實現以下功能：
行動面板顯示所有可執行的行動。

#### Scenario: 行動按鈕
- 解鎖建築按鈕：「解鎖建築（需要 20 靈石）」
- 種植花魂按鈕：「種植花魂（消耗 2 行動力）」
- 澆灌花魂按鈕：「澆灌花魂（消耗 1 行動力，5 絳珠）」

#### Scenario: 按鈕狀態
- 資源或行動力不足時，按鈕禁用（變灰）
- 按鈕禁用時，顯示所需資源或行動力
- 按鈕啟用時，可以正常點擊

#### Scenario: 行動執行
- 點擊按鈕後，執行對應行動
- 行動成功後，顯示成功提示
- 行動失敗後，顯示失敗原因
- UI 更新顯示新的狀態

### Requirement: 遊戲日誌 UI
系統 SHALL 實現以下功能：
遊戲日誌顯示遊戲中的重要事件和操作記錄。

#### Scenario: 日誌顯示
- 顯示遊戲中的重要事件（如「記憶解鎖」「資源獲得」「花魂升級」）
- 日誌可以滾動（如果日誌較多）
- 日誌顯示時間戳或節氣標記（可選）

#### Scenario: 日誌內容
- 記憶解鎖：「記憶「XXX」解鎖！獲得 X 顆絳珠」
- 資源獲得：「獲得 X 顆絳珠」或「獲得 X 顆靈石」
- 行動執行：「種植花魂：黛玉芙蓉魂」「澆灌花魂：消耗 5 顆絳珠」
- 節氣推進：「進入節氣：雨水，行動力恢復到 4 點」

### Requirement: 下一節氣按鈕
系統 SHALL 實現以下功能：
當滿足條件時，顯示「進入下一節氣」按鈕。

#### Scenario: 按鈕顯示條件
- 所有當前節氣的記憶已解鎖（可選）
- 行動力已耗盡（必須）
- 按鈕顯示在明顯位置

#### Scenario: 按鈕功能
- 點擊後，推進到下一個節氣
- 行動力重置為最大值
- 觸發節氣相關事件（如果有）
- UI 更新顯示新節氣

### Requirement: UI 響應式設計
系統 SHALL 實現以下功能：
UI 在不同屏幕尺寸下都能正常顯示。

#### Scenario: 桌面端顯示
- UI 元素清晰可見
- 按鈕大小適中，易於點擊
- 文字大小適中，易於閱讀

#### Scenario: 移動端顯示（可選）
- UI 元素適應小屏幕
- 按鈕大小適合觸摸操作
- 文字大小適合移動端閱讀

### Requirement: UI 視覺風格
系統 SHALL 實現以下功能：
UI 使用簡潔的中國風設計，符合遊戲主題。

#### Scenario: 顏色方案
- 使用柔和的顏色，不過度飽和
- 使用繁體中文標籤
- 視覺風格統一

#### Scenario: 圖標和插畫（可選）
- 使用 SVG 圖標（如絳珠、靈石圖標）
- 使用簡潔的插畫（如建築插畫）
- 視覺元素清晰易識別
